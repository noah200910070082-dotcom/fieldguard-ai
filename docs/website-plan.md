# 智護田專屬網站規劃

## 1. 網站定位

網站採用「展示官網 + 作業控制台」雙層定位。

| 使用者 | 最關心的問題 | 對應頁面內容 |
| --- | --- | --- |
| 比賽評審／合作方 | 創新在哪、能否落地 | 首屏價值、閉環流程、技術架構、開源基礎 |
| 農戶／農場管理者 | 哪裡有風險、現在要做什麼 | 地塊熱力圖、分級預警、處置建議、事件紀錄 |
| 研發與運維人員 | 設備是否在線、模型是否可靠 | 感測節點、機器人狀態、風險因子、模型版本與日誌 |

## 2. 網站綱要

### 首頁 `/`

1. 品牌主視覺：一句話說明「提前發現 + 自主處置」。
2. 核心指標：提前 3–7 天、邊緣延遲小於 500ms、24/7 在線。
3. 即時農田：地塊熱力圖、健康度、風險、巡田覆蓋與節藥估算。
4. 智能研判：光譜異常、病斑面積、環境風險與建議動作。
5. 防治閉環：感知 → 決策 → 執行 → 回測。
6. 最小干預矩陣：監測、補拍、抓取／驅離、精準噴灑。
7. 雲邊端架構：端側感知執行、邊側即時推理、雲端長期分析。
8. 開源技術：SO-101 和 LeRobot 的角色與連結。

### 控制台 `/dashboard`（第二階段）

- 農場／田塊切換；
- GIS 地圖與風險熱力層；
- 即時 RGB 畫面、目標框與光譜曲線；
- 待處理預警、任務佇列、處置紀錄；
- 手動接管、急停、禁噴區和安全確認；
- 農藥存量、電量、訊號與設備健康度。

### 分析中心 `/analytics`（第二階段）

- 日／週／月風險趨勢；
- 病種、蟲害和動物入侵分佈；
- 農藥用量、物理清除比例和節省估算；
- 模型準確率、誤報率與人工覆核結果；
- 大模型生成的農戶易讀週報。

### 設備管理 `/devices`（第三階段）

- 固定基站、巡田車、機械臂和噴灑模組；
- 在線狀態、韌體版本、校準和維修紀錄；
- SO-101 關節狀態與 LeRobot 策略版本；
- 安全區域、最大劑量、低電返航等策略。

## 3. UI 視覺規範

- 品牌色：深田野綠 `#18372E`，表達可信、穩定和農業科技。
- 背景色：暖米白 `#F3F1E9`，降低純白介面的冷感。
- 強調色：嫩芽綠 `#C8E165`，用於科技節點與關鍵狀態。
- 風險色：綠、黃、橙、紅只用於狀態，不用作普通裝飾。
- 字體：中文使用 Noto Sans TC，數據使用 DM Mono。
- 介面原則：先顯示「哪裡、風險多高、建議做什麼」，再展開模型細節。

## 4. 功能優先級

### MVP（目前原型）

- 展示首頁；
- 模擬即時儀表盤；
- 地塊選擇與風險聯動；
- 風險滑桿模擬；
- 處置指令回饋；
- 響應式版面。

### V1（接真實資料）

- REST API 讀取農場、地塊與歷史資料；
- MQTT over WebSocket 接收即時感測和設備狀態；
- 使用者登入與角色權限；
- 預警確認、任務下發與審計日誌；
- ECharts／Leaflet 繪製趨勢和地理熱力圖。

### V2（具身智能）

- LeRobotDataset 記錄影像、關節狀態與動作；
- 遙操作收集抓取示範；
- 模仿學習策略版本管理；
- SO-101 手眼標定與抓取回放；
- 人工覆核結果回流訓練資料。

## 5. 建議資料接口

```text
GET  /api/farms
GET  /api/fields/:id/status
GET  /api/fields/:id/risk-history
GET  /api/alerts?status=open
POST /api/missions
POST /api/missions/:id/approve
POST /api/robots/:id/emergency-stop
GET  /api/devices
```

MQTT 主題建議：

```text
fieldguard/{farmId}/sensors/{deviceId}/telemetry
fieldguard/{farmId}/vision/{deviceId}/detections
fieldguard/{farmId}/robots/{robotId}/state
fieldguard/{farmId}/robots/{robotId}/command
fieldguard/{farmId}/alerts
```

## 6. 風險模型到 UI 的映射

`RiskScore = α × S_spectral + β × S_lesion + γ × S_spread + δ × S_env − F_action`

| 分數 | UI 狀態 | 預設動作 |
| --- | --- | --- |
| 0–30 | 綠色／正常 | 常規監測 |
| 30–50 | 黃色／關注 | 加密採樣、派車近距確認 |
| 50–70 | 橙色／警告 | 抓取蟲卵或聲光驅離、通知農戶 |
| 70–100 | 紅色／嚴重 | 精準施藥、緊急通知、立即回測 |

處置後由 `F_action` 暫時降低評分，但仍以新的感測結果驗證是否真正恢復，避免 UI 只因「已下指令」就錯誤顯示健康。

## 7. GitHub 學習路線

1. SO-101：先學物料清單、3D 列印、組裝、電機校準與相機安裝。
2. LeRobot：再學 `Robot` 統一接口、遙操作、資料錄製和 `LeRobotDataset`。
3. 農業任務：錄製固定高度、固定角度的虫卵抓取示範，先縮小場景提高成功率。
4. 策略訓練：用模仿學習得到初版抓取策略，再逐步加入不同光照和葉片姿態。
5. 安全落地：噴藥控制與學習策略分離，必須保留劑量上限、禁噴區、人工急停和審計日誌。

> 注意：SO-ARM100 倉庫目前推薦使用 SO-101；規劃新硬體時不要再從已 deprecated 的 SO-100 文檔起步。
